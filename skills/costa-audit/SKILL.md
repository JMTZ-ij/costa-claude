# Costa Audit — Operational Audit (No Proposal)

You are the audit-only skill for `/costa audit <url>`. Same depth of analysis as `/costa prospect`, but stops short of the tier proposal. Output is `COSTANERA-AUDIT.md`.

## When To Use This vs `/costa prospect`

- `/costa audit` → analyzing a current client, a referred lead, or a prospect where you've already had a conversation and don't need a cold-outreach proposal generated
- `/costa prospect` → cold-outreach scenarios where you want the report to end with a ready-to-send tier proposal

## Procedure

Run the same Phase 1 (Discovery) and Phase 2 (5 parallel agents) defined in `costa-prospect/SKILL.md`. Skip Phase 3.4 (tier proposal generation).

The 5 agents are unchanged:
1. costa-ops-audit (30%)
2. costa-data-stack (20%)
3. costa-automation (20%)
4. costa-ai-fit (10%)
5. costa-decision-maker (20%)

```markdown
# Operational Audit: [Company Name]

**URL:** [url]

**Business Type:** [type]
**Costanera Fit Score: [X]/100 ([grade])**

## What We'd Build First

A focused 2-paragraph recommendation: if this prospect engaged Costanera tomorrow, what is the single most valuable thing to ship in week 1, and why? Reference the specific automation / dashboard / AI integration identified by the agents. Plain language.

---

## Open Questions

A short list (3-5 items) of things that would change the recommendation if known. Examples:
- "Are they on Shopify Plus or standard? Affects the integration approach."
- "Does the COO have budget authority or does the CFO sign off?"
- "Is the existing Zapier setup actively maintained, or is it 'hanging on'?"

These are the questions Eduardo would ask on a discovery call. Including them in the audit shows the prospect we know what we're doing and signals what discovery covers.

---

*Generated by Costanera AI Sales Studio — `/costa audit`*
```

## Cross-Skill Behavior

- If `COSTANERA-PROSPECT.md` already exists for this URL: tell the user, and offer to skip re-running and just produce the audit-only view from the existing analysis.
- After generating, suggest: `/costa proposal <client>` if the user wants to convert the audit into a proposal, or `/costa outreach <prospect>` if they want a cold email built off the audit findings.
